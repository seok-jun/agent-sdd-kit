# 작업 등급 기준

작업 규모와 실패 영향을 함께 보고 등급을 정한다. 등급은 산출물의 양을 결정한다.

## 기본 등급

| 등급 | 일반적인 범위 | 최소 기록 |
|---|---|---|
| Trivial | 오타, 문구, 주석처럼 동작에 영향이 없는 변경 | 변경 이유, diff 확인 |
| Small | 단일 파일 또는 한 계층 안에서 닫히는 낮은 위험의 변경 | 수정 대상, 수정 금지 영역, 간단한 검증 기준 |
| Medium | 둘 이상의 계층 또는 파일 유형이 함께 바뀌는 기능 변경 | 작업지시, 영향 범위, 테스트 기준, 리뷰 항목 |
| Large | 스키마, 외부 계약, 메시지, 금액·권한처럼 실패 영향이 큰 변경 | 전체 명세, 계약·운영 체크리스트, rollback 기준 |
| Epic | 도메인 구조 변경, 신규 기능군, 여러 시스템을 묶는 변경 | 상위 설계, 하위 작업 분해, 통합 검증 계획 |

`Medium` 이상부터 SDD 산출물을 만든다. `Trivial`과 `Small`에는 전체 SDD 구조를 강제하지 않는다.

## Trivial과 Small의 기록 위치

SDD 파일을 만들지 않는다는 뜻은 기록을 남기지 않는다는 뜻이 아니다.

다음 우선순위로 기존 작업 표면에 최소 기록을 남긴다.

1. 사용자가 지정한 Issue, Ticket 또는 PR 본문
2. 저장소에서 이미 사용하는 작업 관리 문서
3. 별도 위치가 없으면 채팅 출력

기록 위치를 만들기 위해 새 Markdown 파일을 생성하지 않는다. `sdd-doc-scaffold`는 Trivial에 변경 이유와 diff 확인 기준을, Small에 변경 목적·수정 대상·수정 금지 영역·검증 기준을 출력하고 멈춘다.

## 자동 격상 조건

다음 중 하나라도 해당하면 최소 등급을 올린다.

| 변경 행위 | 최소 등급 |
|---|---|
| 외부로 나가는 URL, 필수 파라미터, 고정값, 요청·응답 계약 변경 | Medium |
| 인증·권한 또는 개인정보 처리 변경 | Large |
| 메시지 토픽, Consumer, Batch, 재시도 동작 변경 | Large |
| 금액, 정산, 주문, 재고처럼 데이터 정합성이 중요한 경로 변경 | Large |
| DB 스키마 또는 마이그레이션 변경 | Large |
| 운영 환경 설정이나 credential 취급 방식 변경 | Large |

파일 경로가 아니라 실제 행위로 판정한다. 포맷만 달라진 파일은 격상 근거가 아니고, 이름이 평범한 파일이라도 외부 계약을 바꾸면 격상 대상이다.

자동 격상 조건에 걸린 등급을 낮추려면 이유를 제시하고 사용자 확인을 받는다.
